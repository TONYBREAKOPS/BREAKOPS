SHIFTOPS **File:** `shiftops.html` (~9,500 lines) **Purpose:** Real-time shift management and staffing dashboard for daily operations. ## Overview ShiftOps is the primary daily operations tool. It manages employee scheduling across day and night shifts, tracks attendance (present, absent, tardy, partial, trainer, trainee), assigns seat positions, monitors staffing adequacy, handles overtime imports, generates timesheets, and produces printable/exportable reports. ## Main Interface ### Header Controls - **Pay Period (PP)** — Select from PP3 onward (14-day cycles starting 2026-01-10) - **Week** — Week 1 or Week 2 within the pay period - **Day** — Saturday through Friday with the specific date shown - **Shift** — Day Shift (0600–1800) or Night Shift (1800–0600). Auto-selects based on current time - **Panel View** — "All 3 blocks" or individual 4-hour blocks, plus an employee search field ### Options Menu - **Import OT** — Import Microsoft Teams overtime schedule (XLSX or CSV) - **Save XLSX** — Export current day's roster to a styled Excel workbook - **Timesheet** — Generate unofficial timesheets for the full 14-day pay period - **Print Daily** — Portrait letter-size PDF (roster + comments) - **Print Weekly** — 8-page legal-size PDF (regulars + OT for both shifts) - **Download Weekly Excel** — Weekly Excel workbook with staffing data - **Print Excel Legend** — Reference guide for weekly Excel color codes - **Shift Swap** — Swap schedules between two employees - **Push to BreakOPS** — Push staffing data to the BreakOPS system ### Status Indicators - **UNFILLED pill** — Count of unfilled seats + unfilled absence reasons - **Firebase pill** — Connection status - **Sync pill** — Real-time sync status (idle / pending / saving / synced / live / error) ## Shift Board Layout Each shift has three 4-hour blocks displayed as columns. Each column contains stacked cards: 1. **Staffing Numbers** — Hour-by-hour required vs. scheduled headcount (SHORT / STAFFED / OVER) 2. **Calltakers Table** — Name, Role, Seat, Worked, Hrs, Present, Comment 3. **Dispatchers Table** — Same format 4. **PIC Table** — Same format 5. **OTHER Table** — For OTH roles (SD, TRN, B, ADM) 6. **Supervisors Table** — Same format 7. **Absent Table** — Shows absent/tardy/partial with Reason column 8. **OT / Extra Bar** — Add overtime employees with Role and Segment selectors ## How to Use ### Viewing the Daily Roster The app auto-loads employee data from Firebase and auto-selects the current pay period, week, day, and shift. ### Changing Role 1. Click the role bubble (CT/DP/PIC/SUP/OTH) next to any employee name 2. Select the new role — employee moves to the correct table ### Assigning a Seat 1. Click the seat bubble next to an employee 2. Select from available seat assignments ### Editing Worked Time 1. Click the worked time pill (e.g., "6-10") 2. Change start/end times in the floating editor 3. Press Enter or click away — hours recalculate automatically 4. Shorter worked time auto-detects TARDY or PARTIAL ### Marking Attendance | Status | Effect | |--------|--------| | **YES** | Normal present | | **TRAINER** | Present as trainer (green ring) | | **TRAINEE** | Present as trainee (excluded from staffing counts) | | **PARTIAL** | Left early (yellow ring, creates ghost absent row for missed time) | | **TARDY** | Arrived late (red ring, auto-opens time editor, creates ghost absent row) | | **NO** | Triggers absence reason picker. Cascades absence to all later blocks for the day | ### Adding OT / Extra Personnel 1. Use the "Add OT" bar at the bottom of each block column 2. Select employee or type a name, choose Role and Segment (4/4, 1/2, 2/2) 3. Click "Add" — person appears in the appropriate role table with an "OT" badge ### Importing OT from Microsoft Teams 1. Options > Import OT 2. Select XLSX or CSV file from Teams 3. System matches employees by last name and distributes OT across blocks/shifts ### Comments 1. Click the speech bubble icon in the Cmt column 2. Type a comment — auto-saves. Rows with comments show a yellow icon ### Shift Swap 1. Options > Shift Swap 2. Select Employee A and Employee B 3. Check the day(s) to swap, preview, then confirm 4. Active swaps listed with remove buttons. "SWAP" badge appears on swapped names ### Generating Timesheets 1. Options > Timesheet 2. Loads all 14 days of the pay period from Firebase 3. Opens new window with per-employee timesheets including: - Daily in/out times and hours - Badges: OT, ABSENT (with reason), TARDY, PARTIAL, TRAINER, TRAINEE - Attendance Summary (absent days/hours, tardy/partial counts, reason breakdown) 4. Print individual or group timesheets as PDF ### Daily PDF Print Options > Print Daily — Portrait letter PDF with full roster and comments page. ### Weekly PDF Print Options > Print Weekly — 8-page legal PDF covering all 7 days for both shifts. ### Push to BreakOPS Options > Push to BreakOPS — Writes position overrides and OT extras to the BreakOPS Firebase project. ## Auto-Marking Behaviors - **Vacation**: Employees on vacation are auto-marked ABSENT with reason "VACATION" - **On-Site Training**: Auto-marked as TRAINEE - **Off-Site Training**: Auto-marked as OTH with type TRN - Auto-marks only apply if the employee hasn't been manually edited ## Keyboard Shortcuts | Shortcut | Action | |----------|--------| | Alt+1 / Alt+2 / Alt+3 | Switch to block 1, 2, or 3 | | Alt+A | Switch to "All 3 blocks" view | | Enter (on time input) | Commit worked time change | | Escape (on time input) | Revert worked time change | | Space (on custom select) | Toggle dropdown | | Arrow Up/Down (in dropdown) | Navigate options | | Enter (in dropdown) | Select option and auto-advance to next row | | Type-ahead (on dropdown) | Type letters to filter/preview match | | Shift+Enter (on present) | Toggle backwards through statuses | ## Firebase Collections | Collection | Purpose | |------------|---------| | `employees` | Employee roster (name, seniority, position, schedule) | | `daily_edits` | Per-day per-shift edits (roles, seats, presence, OT, comments) | | `Calltaker/Dispatcher/PIC_Adequate_Staffing` | Required headcounts per hour per day | | `Calltaker/Dispatcher/PIC/Supervisor_Seat_Assignments` | Seat/position labels per role | | `Absent_Reason` | Valid absence reason labels | | `rotation_plans` | Vacation and training data per month | | `scheduledChanges` | Future schedule changes | | `shift_swaps` | Active shift swap records per date | | `bugReports` | Bug reports | --- # 3. BREAKOPS — ADMIN PANEL **File:** `admin.html` (~4,300 lines) **Purpose:** Full-featured break scheduling and seat assignment management. ## Overview BreakOPS Admin is a real-time break scheduling system for managing employee breaks across four positions (Calltaker, Dispatcher, PIC, ANET). It uses a turn-based sequential system where each employee picks their break in order, with skip timers and second-chance mechanics. ## Main Sections ### Navigation Controls - **Date Picker** — Select the working date - **Mode** — Days (0600–1800) or Nights (1800–0600) - **Block** — Select 4-hour block - **Search** — Filter employees by name - **Lock/Unlock** — Password-protected toggle for editing ### Dispatcher Channel Assignments Three block cards showing 10+ channel assignments (North, East, South, West, Northwest, Central, DC1, DC2, R1, R2). Assign dispatchers to channels via dropdowns — swapping is supported. ### Position Lineup Tables Four tables (Calltaker, Dispatcher, PIC, ANET) showing: - Employee name, shift, days off, seniority - Position override dropdown (change position temporarily) - Hours dropdown: "4" (full block), "1/2" (first half), "2/2" (second half) - Coverage mini-table showing adequate vs. scheduled per hour ### Break Picker ("PICK YOUR BREAKS") - **DEPLOY BREAKS** — Starts break selection for all positions - **PAUSE ALL** — Pauses all timers - **Timer Duration** — 10sec to 10min (default 5 min) - **Slot Capacity Manager** — Adjust per-slot, per-position capacity limits (default: CT=3, DP=2, PIC=1, ANET=1) For each position: - Sequential turn order based on seniority - **NEXT UP** (yellow) — Current picker - **SKIPPED** (gray) — Timer expired, waiting for second chance - **2ND CHANCE** (orange) — Skipped person getting another turn - Skip timer with red countdown badge - Break time dropdown and seat assignment dropdown ### NOT IN STAFFING LISTS Shows employees set to ABSENT with a "Restore" button. ### Add Extra People Add overtime/fill-in employees with position selector. Extras appear at end of position lists. ### Print Break Sheets Two-page printout — Page 1: Calltaker breaks, Page 2: Dispatcher/PIC breaks with relief table. ## How to Use ### Setting Up for a Shift 1. Select Mode, Date, and Block 2. Roster auto-populates from employee database 3. Make position overrides as needed 4. Add extras (overtime employees) ### Assigning Breaks 1. Click DEPLOY BREAKS (must lock editing first) 2. NEXT UP person selects their seat assignment, then break time 3. If timer expires, person is skipped; next person highlighted 4. Skipped people get a second chance after the jumper picks ### Pre-Selections & Auto-Pick Admins can set pre-selections for employees. When deployed, auto-pick assigns their preferred break/seat automatically. ### Push Seats to ShiftOps Click "PUSH SEATS" to send CT and PIC seat assignments to ShiftOps. ## Firebase Collections Uses **two separate Firebase projects**: **BreakOps Firebase (legitimous-2f2d2):** - `breakScopes/{scopeKey}` — Breaks, assignments, slot caps, timers, skipped turns, pre-selections - `rosterConfig/global` — Position overrides - `appState/current` — Landing view state - `bugReports` — Bug reports **ShiftOps Firebase (commandops-93846):** - Used for PUSH SEATS feature only --- # 4. ROTATIONOPS **File:** `rotationopsv2.html` (~14,500 lines) **Purpose:** Monthly dispatcher channel rotation scheduling with AI-powered fairness optimization. ## Overview RotationOps generates, manages, and publishes monthly rotation schedules assigning dispatchers to 10 radio channels (North, East, South, West, Northwest, Central, DC1, DC2, Relief 1, Relief 2) across six 4-hour blocks per day. The goal is fair channel distribution while respecting dozens of constraints. ## Core Functions ### RANDOMIZE Runs hundreds of full-month generation attempts (~60 seconds). Each attempt fills every 4-hour block across all 10 channels from a shuffled dispatcher pool, respecting shift schedules, vacations, training, weekly caps, and fairness targets. Keeps the best-scoring plan. ### BALANCE Post-optimization pass that swaps channel and CT assignments to improve fairness without introducing violations. ### CHECK VIOLATIONS Scans for rule violations: same-channel-twice in a day, weekly CT/Relief/DC/primary channel caps exceeded. ### CLEAR ASSIGNMENTS Password-protected ("911911") — wipes all assignments for the current month. ## View Tabs ### Day View Single-day view of all 10 channels x 6 blocks. Click assigned names for actions (SWAP, MOVE, MARK ABSENT). Click NEEDED badges to assign from CT/SD/OT pools. Send day/night shifts to ShiftOps. ### Week View 7-day grid with employee search. Export open shifts. ### Month View Full-month calendar with employee search. ### Roster View Per-employee monthly grid split by Day/Night/Split shift. Printable with channel count totals. ### Monthly Exceptions - **Vacations/Exclusions** — Add/delete vacation ranges per employee - **Trainers** — Add training entries with date ranges, block pickers, channel pickers, and per-day overrides ## Export Options - **Export Totals** — Fairness statistics - **Export Excel** — Full-month roster as .xls - **Export CSV** — All assignments with violation markers - **Export Open Shifts** — Teams-compatible .xlsx of unfilled slots - **Push to ShiftOps** — Sync rotation data to ShiftOps daily_edits ## How to Use ### Generating a Schedule 1. Select Month and Year 2. Click RANDOMIZE — waits ~60 seconds for optimization 3. Review violation count and fairness metrics 4. Click BALANCE to further improve 5. Click CHECK VIOLATIONS to audit ### Manual Adjustments - Click any name → SWAP (exchange with another), MOVE (relocate to empty slot), MARK ABSENT - Click NEEDED badge → Assign from CT, SD, or overtime pool - Click Vacation name → Assign to empty slot as overtime ### Remove from Rotation Password-protected. Select employee, set date range — their slots become NEEDED. ## Configuration - **Channels:** 10 main + 7th channel (Friday/Saturday nights only) - **Blocks:** Six 4-hour blocks (0600–0600) - **Weekly caps:** CT 2 blocks, Relief 2 blocks, DC 2 blocks, Primary channel 2 blocks - **SD monthly cap:** 32 blocks (128 hrs) - **Theme:** Dark/Light toggle saved in localStorage ## Firebase Collections - `rotation_plans` — Complete monthly rotation (plan, vacations, trainers, overrides) - `employees` — Dispatcher roster (filtered to DISPATCHER position) - `daily_edits` — ShiftOps integration for push - `scheduledChanges` — Mirror training entries - `Dispatcher_Seat_Assignments` — Seat name casing for push - `bugReports` — Bug reports --- # 7. MANDATEOPS **File:** `mandateops.html` **Purpose:** Overtime mandate ranking, acceptance/refusal tracking, and refusal form generation. ## Overview MandateOps generates ranked lists of employees eligible for mandatory overtime, records mandate acceptance/refusal, and produces formal documentation. ## Controls - **Date Picker** — Select mandate date - **Shift Selector** — Day (0600–1800) or Night (1800–0600) - **Mandate Block** — 4-hour time block within the shift - **Generate List** — Build the ranked eligibility list - **Import OT** — Upload Teams export for OT hour tracking - **Print List** — Print all three role lists ## Role Tabs CALLTAKER | DISPATCHER | PIC — each with independent mandate rankings. ## Ranking Algorithm Employees ranked by: 1. **Shortest work day** (ascending) — fewest total hours that day 2. **Least OT hours in last 7 days** (ascending) 3. **Highest seniority number** (descending) — for ties ## Eligibility Rules - Shift must end exactly when mandate block starts - Must be marked Present - Not already on OT overlapping the block - Would not exceed 16-hour work day cap ## How to Use ### Generating a Mandate List 1. Select date, shift, and mandate block 2. Import OT from Teams if available (for accurate weekly OT tracking) 3. Click "Generate List" 4. System loads attendance, checks OT history, ranks eligible employees ### Recording Actions - Click **Mandate** to accept an employee for OT - Click **Refuse** to record a refusal - Click **Undo** to reverse (today's actions only) - Click **Print Refusal** to generate a formal refusal form with signature lines ### Mandate Log Right-side panel shows real-time log of all mandate/refusal actions. Clear Log is password-protected ("166637805"). ### Cross-Qualification - CT positions: check CT first, then DP, then PIC - DP positions: check DP first, then PIC - PIC positions: PIC only ## Firebase Collections - `employees` — Employee roster - `daily_edits` — Attendance data - `mandates` — Mandate action log - `bugReports` — Bug reports --- # 11. DECISIONOPS **File:** `decisionops.html` (~3,100 lines) **Purpose:** Structured meeting decision engine with real-time voting. ## Overview DecisionOps guides teams through a structured decision-making process: organized topic discussion, timed speakers, option narrowing, participant voting, and action item tracking. All synchronized in real-time via Firebase. ## Meeting Lifecycle **Create → Setup → Start → Active → Stop/Resume → End** ## Topic Phases ### WAITING Default state. Editable title, description, submitted-by, notes, time limit. Reorderable with arrow buttons. ### DISCUSS - Timer counts down from allocated time - Track active speakers with per-person countdown timer - Propose options/solutions - Shared notes section - Can declare consensus, table the topic, or advance to narrowing/voting ### NARROW (3+ options) Facilitator selects 2–4 finalist options from all proposals (checkbox interface). ### VOTE - Real-time per-person voting board - Facilitator clicks options to assign votes on behalf of participants - Shows vote count and percentage live - Facilitator closes voting to reveal results - Tied results → facilitator breaks tie ### DECIDED Winner displayed with vote breakdown, per-person vote history, actual vs. allocated time, participation stats, and action items with owner/due date. ## How to Use ### Creating a Meeting 1. Enter meeting title, click "Start Meeting" 2. Add topics (title, description, time limit) 3. Add participants (search employees, or type "SUPERVISORS"/"DISPATCHERS" for bulk add) 4. Click START MEETING ### Running Discussion 1. Click a WAITING topic to begin DISCUSS phase 2. Click participant names to track speaker time 3. Propose options in the input field 4. Click "Advance to Vote" (2 options) or "Advance to Narrow" (3+ options) 5. Or click "Declare Consensus" if everyone agrees ### Voting 1. Facilitator or participants click option buttons to vote 2. Click "Close Voting & Reveal" to show results 3. If tied, facilitator selects winner 4. Click "Finalize Decision" ### Action Items In DECIDED phase, add action items with task description, owner, and due date. ### Summaries - **COPY** — Plain-text meeting summary to clipboard - **PRINT** — Formatted print-ready summary in new window ## Firebase Collections - `decisionMeetings` — All meeting data (topics, votes, participants, action items) - `employees` — Employee search for participant adding - `appSettings` — Saved lead teams (doc: `decisionLeads`) --- # 12. Write operations use debounced auto-save (typically 1.5–3 seconds) to prevent excessive writes.
